## 1. Dashboard
The **Dashboard** is your primary landing page. It provides:
- **System Status**: See if the device server is online or offline.
- **Server Control**: Start or stop the communication server using the button at the top.
- **Quick Links**: Access recently added patients or pending reports.

## 2. Patient Management
Manage patient records efficiently:
- **Add Patient**: Register new patients with demographic information (Name, ID, Gender, Age).
- **Search**: Use the search bar to find existing patients by name or ID.
- **History**: View all previous tests and reports associated with a patient.

## 3. CBC Reports
The **CBC Report** view is dedicated to Complete Blood Count results:
- **Result Entry**: View data received from automated hematology analyzers.
- **Validation**: Review and validate results against normal ranges.
- **Printing**: Generate professional PDF reports for patients and physicians.

## 4. Send Out Management
For tests that are processed at external laboratories:
- **Order Tracking**: Log tests sent to third-party labs.
- **Status Updates**: Mark tests as "Pending", "Sent", or "Received".
- **Result Recording**: Enter results once they are returned from the external laboratory.

## 5. Live Stream
The **Stream** view allows real-time monitoring of device communications:
- **HL7 Messages**: View incoming and outgoing HL7 strings.
- **Connection log**: Monitor which devices are currently connected to the server.
- **Debugging**: Useful for verifying that data is being received correctly from analyzers.

## 6. Settings
Customize the application to your laboratory's needs:
- **General**: Change the Application Name and Slogan.
- **Network**: Configure the Server IP and Port for device communication.
- **Database**: Update connection strings for your SQL database.
- **Appearance**: Toggle between Light and Dark modes.
